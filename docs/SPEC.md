# 仕様書 — andw-thumbnail-column

## 要求定義

| # | 項目 | 内容 |
|---|------|------|
| 1 | 対象ユーザー | WordPress 管理者・編集者 |
| 2 | 起点の課題 | 投稿・固定ページの一覧画面ではサムネイルが表示されず、中身を開かないと画像が確認できない |
| 3 | 理想の行動変化 | 一覧画面を見るだけでサムネイルが分かり、記事の識別・選択が素早くできる |
| 4 | 事業上の狙い | 管理画面の作業効率を上げる。自社・クライアント両方のサイトで使い回せる汎用プラグインにする |
| 5 | 成功指標 | 一覧画面にサムネイル列が表示される。投稿・固定・カスタム投稿タイプに対応。有効/無効を投稿タイプごとに切り替えられる |

---

## 要件定義

### 機能要件

#### 1. サムネイル列の表示
- 投稿一覧・固定ページ一覧・カスタム投稿タイプ一覧の **最右列** にサムネイル列を追加
- 列幅: 行の高さに合わせた正方形（約60×60px）
- 画像は `get_the_post_thumbnail()` で取得（thumbnail サイズ）
- サムネイルなしの場合: グレーの placeholder（CSS で表現、画像ファイル不要）

#### 2. 投稿タイプごとの有効/無効切り替え
- WordPress 管理画面 → 設定 → サムネイル列 に設定ページを作成
- WordPress に登録されている投稿タイプ（`show_ui = true`）を自動検出して一覧表示
- 各投稿タイプにチェックボックスで有効/無効を設定
- デフォルト: `post`（投稿）と `page`（固定ページ）を有効

#### 3. 非機能要件
- WordPress 6.0 以上対応
- PHP 8.0 以上対応
- 他プラグインとの競合を避けるため、プレフィックス `andw_tc_` を使用
- アンインストール時にオプションを削除

## 画面仕様

### 一覧画面（投稿・固定ページ・カスタム投稿タイプ）

```
| チェック | タイトル | ... | 日付 | サムネイル |
|---------|---------|-----|------|-----------|
|  □      | 記事A   | ... | 2024 | [画像60px] |
|  □      | 記事B   | ... | 2024 | [グレー]   |
```

- 列ヘッダー: 「サムネイル」
- 画像スタイル: `width:60px; height:60px; object-fit:cover;`
- placeholder スタイル: `width:60px; height:60px; background:#ddd;`

### 設定画面（設定 → サムネイル列）

```
サムネイル列を表示する投稿タイプ:

[✓] 投稿 (post)
[✓] 固定ページ (page)
[ ] 商品 (product)
[ ] お知らせ (news)

[保存]
```
