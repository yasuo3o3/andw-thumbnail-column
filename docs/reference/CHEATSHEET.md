```
┌──────────────────────────────────────────

├──────────────────────────────────────────
│  B. ペルソナレビュー（Sonnet 5並列）【任意】

├──────────────────────────────────────────

├──────────────────────────────────────────
│  E. コードレビュー（Sonnet 5並列 → Opus統合）
│  ──────────────────────────────────────────
│  Sonnet: beginner / crawler / security / wp / breaker
│  → Opus統合 → BUG/FIX/IMPリスト → 修正プロンプト

├──────────────────────────────────────────
│  E'. 修正サイクル（Sonnet→Opus確認）
│  ──────────────────────────────────
│  Sonnet修正 → Opus確認 → BUG残なら Opus直接修正
├──────────────────────────────────────────
│  F. テスト・検証（Opus + オーナー）
│  ──────────────────────────────────
│  /create-test-issues → 管理画面テスト(やすお) + フロントテスト
│  → オーナーのユーザーテスト（要求定義の検証）
│
│  📋 /estimate（完成後） /test-report /acceptance-checklist
│     /user-guide /spec-doc

```

|------|------|
| **Tab** | 許可ダイアログで追加指示を入力 |

## 要求定義 5項目（毎回確認）

1. **対象ユーザー** — 誰の困りごと？
2. **起点の課題** — 何が未解決？
3. **理想の行動変化** — どうなってほしい？
4. **事業上の狙い** — なぜ今これを？
5. **成功指標** — 何で成功を測る？


## 既存リポジトリに WPCS を導入する手順

1. テンプレートから `composer.json` と `phpcs.xml.dist` をコピー
2. `phpcs.xml.dist` を編集:
   - `PLUGIN-TEXT-DOMAIN` → プラグインのテキストドメイン
   - `andw_` / `Andw_` / `ANDW_` → プラグイン固有のプレフィックス
3. 実行: `composer install && composer phpcs`

**Claude に依頼するときのコピペ文:**
> このリポジトリに WPCS（WordPress Coding Standards）を導入してください。
> `c:\andW\andw-template` の `composer.json` と `phpcs.xml.dist` を参考に、
> **既存の composer.json がある場合は require-dev にマージ**、なければコピーしてください。
> phpcs.xml.dist のテキストドメインとプレフィックスをこのプラグインに合わせて変更、
> composer install を実行して composer phpcs でチェックを通してください。
> エラーが出たら修正もお願いします。
