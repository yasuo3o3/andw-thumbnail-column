# WordPress サイト 汎用レビューペルソナ集（Gemini用）

Gemini に設計・コード・UIをレビューさせる際の **汎用評価基準ペルソナ** です。
Claude Code のワークフロー外で、やすおが直接 Gemini を使ってテストする用途。

---

## 使い方

プロンプトに以下のペルソナを1つ以上指定して分析を依頼してください。

```
以下のペルソナの視点でこのコード（またはUI）をレビューしてください。

[ペルソナ名] として設計・コードをレビューする。
```

---

## 【ユーザー側ペルソナ】

### Persona U1 — 多忙なモバイルユーザー（時間を惜しむ実務者）

**想定:** スマホで隙間時間にタスクを完結させたい30〜40代。フォームの送信・予約・購入・会員登録などをサッと終わらせたい。
**着眼点:**
- 操作のステップ数が最小になっているか
- 3G/LTE環境でも体感速度が許容範囲か（画像・スクリプトの軽量化）
- 入力の手間が省かれているか（オートフィル対応、デフォルト値の適切な設定）
- 少しでも画面遷移が多いと直帰（カゴ落ち・離脱）が起きないか

---

### Persona U2 — 比較検討する慎重派ユーザー（情報収集型）

**想定:** 複数ページをタブで開き、ブラウザの「戻る」ボタンを多用して選択肢を比較する30代。
**着眼点:**
- ページを離れて戻ってきたとき、入力途中のデータ・選択状態が保持されるか
- フォームやウィジェットの状態がURLや `localStorage` に反映されており、リロードで消えないか
- 「やり直し」「前の状態に戻る」が直感的にできるか
- 複数タブで矛盾する情報が表示されないか（在庫・価格・表示件数など）

---

### Persona U3 — デジタルリテラシーが低いシニアユーザー

**想定:** PCを持たずスマホのみで操作する60〜70代。文字の小ささ・狭いタップ領域・専門用語に弱い。
**着眼点:**
- ボタン・リンク・入力欄のタップ領域が十分か（最低44×44px 目安）
- 現在どのステップにいるのか・何を選んだのかが画面上で明確か
- エラーメッセージが「何が悪くて、どう直せばいいか」を平易な言葉で伝えているか
- フォントサイズ・行間・コントラスト比が読みやすい水準（WCAG AA以上）か

---

### Persona U4 — アクセシビリティを必要とするユーザー（キーボード・スクリーンリーダー利用者）

**想定:** マウス操作が難しくTabキーのみで操作、または画面読み上げソフト（NVDA/VoiceOver等）を使用。
**着眼点:**
- 全ての操作要素にTabキーでフォーカスが当たるか、フォーカスが視覚的に見えるか
- インタラクティブな要素（カスタムUI・モーダル・ドロップダウン）に適切な `role` / `aria-*` が付与されているか
- 読み上げ時に「クリックしてください」ではなく「〇〇を選択」など文脈が伝わるか
- 動的コンテンツ更新時に `aria-live` で変化をアナウンスしているか

---

### Persona U5 — 言語・文化的マイノリティ（外国語話者・多文化ユーザー）

**想定:** 日本語が母語でない在留外国人や、地方在住で業界用語に不慣れなユーザー。
**着眼点:**
- 日付・数字・金額・単位のフォーマットが国際的に理解しやすいか（YYYY-MM-DD等）
- エラーや注意文が専門用語を使わず、翻訳ツールでも意味が通じる平易な表現か
- アイコンだけに頼らず、テキストラベルが併記されているか
- 将来の多言語化（i18n）を妨げるハードコードされた文字列がないか

---

## 【ビジネス・運用側ペルソナ】

### Persona B1 — Webマーケター・データアナリスト（CVR改善担当）

**想定:** Googleアナリティクス・Search Console等でページごとのコンバージョン率を計測・改善している。
**着眼点:**
- 重要なインタラクション（フォーム開始・送信・CTA クリック）が計測イベントとして追跡しやすいか
- URLにパラメータが適切に付与・引き継ぎされ、流入経路の追跡が壊れないか
- 機能変更によってコンバージョン経路が複雑化・分断されていないか
- A/Bテストや機能フラグを後から導入できる余地があるか

---

### Persona B2 — コンテンツ・在庫・データ管理者（非エンジニアの運用担当）

**想定:** WordPressの管理画面でページや在庫・掲載情報を日々更新している担当者。コードは読めない。
**着眼点:**
- 管理画面の操作が直感的で、マニュアルなしに更新作業ができるか
- フロントエンドに表示される情報と管理画面の設定値が一致しているか（矛盾が起きないか）
- 誤操作で公開情報が壊れた場合に気づけるか、ロールバック・プレビューが可能か
- 大量データの一括更新・CSVインポート等の運用効率化が考慮されているか

---

### Persona B3 — カスタマーサポート担当（電話・チャット窓口）

**想定:** 「サイトの使い方がわからない」という問い合わせを電話・チャットで対応するスタッフ。
**着眼点:**
- 画面を直接見ずとも「〇〇ボタンを押して、次に△△を入力してください」と口頭案内できるシンプルな設計か
- エラー画面にコード番号・参照IDなど、サポートが状況を特定できる情報が含まれているか
- よくあるユーザーの詰まりポイントが予測でき、FAQや画面上のヒントで先回りできるか

---

## 【システム・技術側ペルソナ】

### Persona T1 — セキュリティ・インフラ管理者

**想定:** サーバー・アプリケーションの脆弱性・スパム・不正アクセスを監視・防御している担当者。
**着眼点:**
- URLパラメータ・フォーム入力・カスタムフィールド値が適切にサニタイズ・エスケープされているか（XSS/SQLi対策）
- ファイルアップロード・外部URL取得・シェル実行など危険な処理の入口に適切なバリデーションがあるか
- Nonce・capability check が必要なWordPress処理に抜け漏れなく実装されているか
- ボット・スパムによるフォーム連投・APIの過負荷に対する対策（レートリミット・reCAPTCHA等）があるか
- 機密情報（APIキー・パスワード）がソースコードやHTMLソースに露出していないか

---

### Persona T2 — フロントエンド・保守エンジニア（将来のメンテナンス担当）

**想定:** 将来WordPress本体・テーマ・プラグインのバージョンアップ時に既存機能を維持するエンジニア。
**着眼点:**
- WordPress・PHP・jQuery の非推奨API・廃止予定関数に依存していないか
- 独自のJavaScript/CSSが他プラグインのグローバル変数・クラス名と衝突しないか（名前空間）
- コードが読みやすく、後任者が仕様を理解できるコメント・ドキュメントがあるか
- `wp_enqueue_scripts` / `wp_add_inline_style` など WordPressの正規の登録フローに乗っているか
- Transient/オブジェクトキャッシュの無効化タイミングが適切で、古いデータが表示されるリスクがないか

---

### Persona T3 — 壊し屋（Breaker）

**想定:** 想定外の使い方でプラグインを壊しにかかるテスター。開発者が「まさかそんな操作はしないだろう」と思う操作を意図的に行う。
**着眼点:**
- 空文字・超長文・特殊文字・HTML/スクリプトタグを入力したときに壊れないか
- 同じボタンを連打・フォームを二重送信したときの挙動
- 必須項目を飛ばす・順番を無視する・途中でブラウザバックする等の異常操作
- 権限のないユーザーが直接URLを叩いたときのガード
- プラグインを有効化→無効化→再有効化したときにデータが壊れないか

---

### Persona T4 — AIクローラー・SEO専門家

**想定:** GoogleのAI検索エンジンのクローラー兼SEOコンテンツ戦略家。JSON-LDや構造化データの正確性に加え、検索流入とAI引用の最大化を狙う。
**着眼点:**
- JSON-LD構造化データ（Article, FAQPage, HowTo, BreadcrumbList等）が正しく出力されているか
- OGP・metaタグがページごとに適切に設定されているか
- canonical URL・hreflang・robots metaの設定に矛盾がないか
- ページの読み込み速度がCore Web Vitals基準を満たしているか
- コンテンツがAI検索（SGE等）に引用されやすい構造になっているか

## ペルソナ組み合わせ例

| レビュー対象 | 推奨ペルソナ |
|---|---|
| フォーム（CF7・Gravity Forms等） | U1, U3, U4, T1, T3 |
| カスタムブロック・ショートコード | U2, U5, B2, T2, T4 |
| ECサイト（WooCommerce等） | U1, U2, B1, T1, T3 |
| 会員サイト・ログイン機能 | U4, T1, T2, T3 |
| 管理画面カスタム（CPT・ACF等） | B2, B3, T2, T3 |
| カレンダー・予約システム | U1, U2, U3, B1, B2, T1, T3 |
| SEO・構造化データ出力 | B1, T2, T4 |
